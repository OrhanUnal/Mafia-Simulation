github kullanmayi bildiginizi biliyorum ama hnp ile kullanirken bazi ozellikleri kullanabilmek icin bu dosya lazim 

1. Branch kullanimi cok onemli, hackh n plan (HNP) de bulunan her bir task icin branch acmaniz ve maine neredeyse hicbir zaman commit atmamaniz gerekiyor. Branch kullanimi cok onemli cunku iyi bir pipeline gerekli ve diyelim ki movement branchiniz var ve basic movement yaptiniz sonra pull request attiniz diger sprintte ise complex movement yapmaniz gerekti o zaman ayni branchi kullanabilirsiniz ama onun disinda neredeyse her zaman her bir task icin branch acmaniz lazim 
2. branch isimleri cok onemli hepsinin formati su sekilde olmali: hnp-*taskin numarasi* *taskin adi*. Ornegin hnp-5 Basic Movement.
3. attiginiz commitlerin de ismi onemli. Bu commitlerin hnp de gozukmesi icin su formati korumalisiniz: hnp-*taskin numarasi* *commitin adi*. Ornegin hnp-5 Basic Movement Final Commit.

bunun disinda HNP de planned kisminda baska birisi tarafindan alinmamis veya almak istediginiz bir task gorurseniz eger alan kisi varsa rica ederek yoksa duz kendinizi assigner olarak ilan edip progress kismina tasimaniz ve isi yapmaya baslamaniz lazim. Sonra isiniz bitince testing kismina atmaniz ve gerekli kisi veya kisiler de testi yaptiktan sonra pull request gondermeniz gerekmektedir. Pull requesti attiginiz zaman taski done kismina tasiyabilirsiniz.
